### 分析

- 6.6k stars/周 表明社区对推理优化的强烈需求

### 叙事判断
> **趋势确认 / 加速信号**：LLM 推理优化从"学术研究"进入"工程实用化"阶段。投机解码可能成为 LLM 部署的标准组件之一。

## 四、信号三：Agent 基础设施层范式转变 —— 从"黑盒执行"到"可观测工程"

### 数据支撑
| 项目 | 链接 | Stars | 关键特性 |
|------|------|-------|---------|
| **shepherd-agents/shepherd** | GitHub | 1,459（周） | 可逆 Git 式追踪，CoW fork 比 docker 快 5x，95% KV-cache 重用 |
| **mereyabdenbekuly-ctrl/clodex-ide** | GitHub | 835（日增） | 本地优先、零信任 Agentic IDE |

### Shepherd 的核心创新
1. **执行可逆**：将 Agent 的执行过程转化为类似 Git 的追踪，支持观察、Fork、回放、回滚
2. **Copy-on-Write Fork**：比 docker commit 快约 5 倍
3. **KV-cache 重用**：回放时约 95% 的 KV-cache 可复用
4. **Meta-Agent 架构**：专为"元 Agent 监督、优化、训练其他 Agent"而设计

### 叙事判断
> **新兴趋势**：Agent 正在从"单次对话式任务执行"演变为 **"可观测、可回放、可优化的系统工程化"** 范式。Shepherd 代表了 Agent 基础设施从"玩具"到"生产级"的关键一步。

## 五、信号四：测试时训练（TTT）跨领域迁移 —— 从视觉到机器人

| 论文 | 作者 | 发布时间 | 关键点 |
|------|------|---------|--------|

### 分析
- TTT（Test-Time Training）此前主要应用于视觉领域（如视频理解、视图合成）
- RoboTTT 将 TTT 引入机器人控制，在测试时动态适应新的环境变化
- 这是一个**跨领域迁移信号**——TTT 正在成为基础模型时代的普适性范式
- 同一批 arXiv 论文中还有 **Online Neural Space Time Memory**（将 TTT 用于动态视角合成），进一步印证 TTT 的扩散趋势

### 叙事判断
> **新兴趋势 / 扩散信号**：TTT 从视觉领域向机器人、3D 渲染等领域迁移，可能成为继 RLHF、RAG 之后的又一通用性 AI 训练/推理范式。

### 分析
- **数据投毒论文**指出：与以往在 Wikipedia 等受控数据集上的研究不同，真实世界的预训练数据来源极其庞杂，存在通过计算宣传（computational propaganda）大规模投毒的攻击面
- **物理安全论文**揭示了一个根本性问题：LLM 输出的"文本安全"不代表"行动安全"——一个文字上无害的指令在物理世界中执行时可能造成危害
- **成本感知评估论文**提出：仅看"成功率"不足以评估安全 Agent，每个推理步骤和工具调用都有成本

### 叙事判断
> **趋势加速信号**：AI 安全研究正从"红队/越狱"扩展到"预训练供应链安全、物理世界安全、评估范式改革"等多个新维度。这是一个结构性的研究方向迁移。
